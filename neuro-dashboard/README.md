# لوحة المالية والإحصائيات - جراحة الأعصاب

لوحة تحكم مالية كاملة لطبيب جراحة أعصاب: تتبع الراتب والاستدعاءات ونسب الأدوية والنسب الثابتة وقسم العمليات والإجازات، مع تخزين محلي آمن على الجهاز عبر **Dexie.js / IndexedDB** ونسخ احتياطي اختياري على **Google Drive**.

## التقنيات المستخدمة
- React 18 + Vite
- Tailwind CSS (دعم كامل لـ RTL)
- Recharts للرسوم البيانية
- Lucide React للأيقونات
- Dexie.js + dexie-react-hooks للتخزين التفاعلي في IndexedDB
- Google Identity Services + Drive API v3 للمزامنة السحابية
- Web Crypto API (AES-GCM) للنسخ الاحتياطي المشفّر - بدون أي مكتبة تشفير خارجية

## التشغيل محليًا
```bash
npm install
npm run dev      # يفتح التطبيق على http://localhost:5173
```
للنشر:
```bash
npm run build     # ناتج البناء في مجلد dist/
npm run preview   # لمعاينة نسخة البناء النهائية
```
التطبيق ثابت بالكامل (Static SPA) ويمكن نشره على أي استضافة مثل Vercel أو Netlify أو GitHub Pages أو حتى فتحه مباشرة من ملف `dist/index.html` بعد البناء.

> **ملاحظة عن التخزين**: البيانات تُخزَّن في IndexedDB الخاصة بالمتصفح/الجهاز، وهي معزولة عن كل "نطاق" (origin). إذا تم نشر التطبيق على رابط معيّن واستُخدم منه، فستبقى البيانات مرتبطة بهذا الرابط على هذا الجهاز/المتصفح بالتحديد. لذلك يُنصح بأخذ نسخة احتياطية دوريًا (محليًا أو على Google Drive) خصوصًا قبل تغيير المتصفح أو الجهاز.

## منطق السنة المالية
السنة المالية تبدأ **1 أبريل** وتنتهي **31 مارس**. كل التجميعات السنوية (الرسوم البيانية، حلقة السنة المالية في الرئيسية، شاشة الإجازات) تستخدم هذا التقويم حصريًا، وهو محسوب في `src/db/fiscalYear.js`.

## قواعد الإجازات والمكافآت
معرّفة في `src/db/calculations.js`:
1. أي يوم يقع داخل فترة إجازة مسجّلة يُستثنى من حساب الاستدعاءات ونسب الأدوية لذلك اليوم، ويُحسب فقط الراتب الأساسي. إذا كان الشهر بالكامل داخل إجازة، تُحسب النسب الثابتة الشهرية كصفر لذلك الشهر أيضًا.
2. إذا انتهت سنة مالية كاملة (تجاوز تاريخ اليوم 31 مارس من تلك السنة) بدون أي إجازة مسجّلة فيها، تُضاف تلقائيًا مكافأة بقيمة راتب شهر واحد (متوسط الرواتب المسجَّلة في تلك السنة) إلى إجمالي تلك السنة.

## إدخال البيانات
كل قسم من المدخولات (الاستدعاءات، نسب الأدوية) يدعم وضعين:
- **إدخال يومي**: تاريخ + قيمة لكل يوم.
- **إدخال إجمالي شهر سابق**: لتسجيل بيانات تاريخية بسرعة عن طريق إدخال إجمالي الشهر دفعة واحدة.

## النسخ الاحتياطي المحلي
من صفحة **"المزامنة والنسخ الاحتياطي"**:
- أدخل كلمة مرور للتشفير (لا تقل عن 4 أحرف، ويُفضّل أن تكون أقوى من ذلك).
- اضغط **"تحميل نسخة احتياطية"** لتنزيل ملف `.json` مشفّر بالكامل (AES-GCM) يحتوي على كل بيانات التطبيق.
- لاستعادة البيانات: اختر الملف من **"استيراد من ملف"** وأدخل نفس كلمة المرور المستخدمة عند التصدير. الاستعادة تُفرغ الجداول الحالية وتستبدلها بمحتوى الملف، فاحرص على أخذ نسخة احتياطية حديثة قبل الاستعادة إذا كانت لديك بيانات لا تريد فقدها.

## تفعيل مزامنة Google Drive
المزامنة تستخدم صلاحية `drive.file` فقط، أي أن التطبيق لا يستطيع رؤية أي ملف آخر في Drive الخاص بك، فقط الملف الذي ينشئه هو بنفسه (`neuro-finance-backup.json`).

خطوات الإعداد (تُنفَّذ مرة واحدة من مطوّر/مالك التطبيق):
1. اذهب إلى [Google Cloud Console](https://console.cloud.google.com/) وأنشئ مشروعًا جديدًا (أو استخدم مشروعًا موجودًا).
2. من "APIs & Services" → "OAuth consent screen": اضبط نوع التطبيق كـ External (أو Internal إذا كنت تستخدم Google Workspace)، وأضف بريدك كمستخدم تجريبي إذا بقي التطبيق في وضع Testing.
3. من "APIs & Services" → "Credentials" → "Create Credentials" → "OAuth client ID":
   - نوع التطبيق: **Web application**
   - أضف رابط التطبيق (مثل `http://localhost:5173` للتطوير، ورابط النشر الفعلي) تحت **Authorized JavaScript origins**.
4. فعّل **Google Drive API** من "APIs & Services" → "Library".
5. انسخ الـ Client ID الناتج، وأنشئ ملف `.env` في جذر المشروع (نسخة من `.env.example`) وضع فيه:
   ```
   VITE_GOOGLE_CLIENT_ID=ضع-المعرّف-هنا.apps.googleusercontent.com
   ```
6. أعد تشغيل `npm run dev` / أعد البناء بـ `npm run build`.

بعد ذلك ستظهر زر **"الاتصال بـ Google Drive"** فعّالًا في صفحة المزامنة، وسيمكنك رفع/استرجاع النسخة الاحتياطية المشفّرة من حسابك على Drive.

## بنية قاعدة البيانات (Dexie / IndexedDB)
معرّفة في `src/db/db.js`:
- `monthlySalaries` - الراتب الأساسي لكل (سنة، شهر)
- `onCallEntries` - الاستدعاءات اليومية/الإجمالية، مع `hospital: 'old' | 'new'`
- `medicationEntries` - نسب الأدوية
- `fixedPercentages` - النسب الثابتة الخمس لكل (سنة، شهر)
- `operations` - سجل العمليات الجراحية
- `operationTypes` - القائمة العامة لأنواع العمليات (الإكمال التلقائي القابل للإضافة)
- `leavePeriods` - فترات الإجازات/السفر
- `appMeta` - بيانات تشغيلية عامة (مثل وقت آخر مزامنة)

## ملاحظات أمنية
- كلمة مرور النسخة الاحتياطية لا تُخزَّن في أي مكان؛ إذا فُقدت، لا يمكن فتح أي نسخة احتياطية سابقة بها. احتفظ بها في مكان آمن.
- رمز الوصول لـ Google (Access Token) يبقى في ذاكرة المتصفح فقط طوال الجلسة الحالية ولا يُخزَّن في IndexedDB أو أي تخزين دائم؛ ستحتاج لإعادة الاتصال بعد إغلاق التطبيق أو تحديث الصفحة.
